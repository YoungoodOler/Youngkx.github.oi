## 1. 当前网站结构

网站目前使用以下技术：

- Next.js：页面、路由和静态构建；
- React：页面组件；
- Framer Motion：滚动、卡片和过渡动画；
- Three.js：主页背景星球与交互粒子场；
- Cloudflare Workers：正式部署、静态文件托管和域名重定向；
- GitHub：保存源代码和版本历史。

访问关系如下：

```text
访问 youngkx.cn
        ↓ 308 永久重定向
访问 www.youngkx.cn
        ↓
Cloudflare Worker youngkxblog
        ↓
读取 Next.js 构建生成的 out/ 静态文件
```

GitHub Pages 已停用。仓库中不再包含旧 Hexo 主题、GitHub Pages 工作流、`CNAME`、`.nojekyll` 或旧静态服务器脚本。

## 3. 第一次配置电脑

### 3.1 安装必需软件

需要安装：

1. Node.js 22 或兼容版本；
2. Git；
3. 一个代码编辑器，例如 Visual Studio Code；
4. 可访问 GitHub 和 Cloudflare 的网络环境。

检查是否安装成功：

```powershell
node --version
npm.cmd --version
git --version
```

### 3.2 获取项目

```powershell
git clone git@github.com:YoungoodOler/Youngkx.github.oi.git D:\youngkx
Set-Location D:\youngkx
npm.cmd install
```

如果使用 HTTPS：

```powershell
git clone https://github.com/YoungoodOler/Youngkx.github.oi.git D:\youngkx
```

### 3.3 登录 Cloudflare

第一次部署前运行：

```powershell
npx.cmd wrangler login
```

浏览器会打开 Cloudflare 授权页面。确认授权后回到终端即可。

不要把 Cloudflare Token、GitHub Token、账号密码或 `.env` 文件提交到仓库。

进入新的项目根目录后运行：

Set-Location D:\youngkx

## 5. 最常用的维护流程

每次修改建议按照以下顺序：

```powershell
# 1. 获取远端最新代码
git pull --ff-only

# 2. 启动本地预览
npm.cmd run dev

# 3. 修改并在浏览器检查

# 4. 执行类型检查
npm.cmd run check

# 5. 执行生产构建
npm.cmd run build

# 6. 查看将要提交的文件
git status
git diff

# 7. 提交到 GitHub
git add 文件名
git commit -m "简要说明本次修改"
git push

# 8. 部署到 Cloudflare
npx.cmd wrangler deploy
```

重要：当前 GitHub 与 Cloudflare 没有自动构建连接。`git push` 只负责保存代码，正式网站更新还需要执行 `npx.cmd wrangler deploy`。

找到：

const introQuotes = [

posts.slice(0, 3)

## 7. 修改或新增文章

### 7.1 推荐方式：使用 Markdown

新文章统一放在 `content/posts/`。先复制 `content/posts/_template.md`，再把副本改成普通文件名，例如 `my-ai-note.md`。以下划线开头的模板不会发布。

每篇文章顶部必须带有 YAML 信息：

```markdown
---
title: "我的 AI 笔记"
date: "2026-07-23"
slug: "my-ai-note"
tags:
  - AI
  - 随笔
card: ai
excerpt: "这是一篇用于说明 Markdown 发布流程的文章。"
draft: false
---

## 正文标题

从这里开始使用 Markdown 编写正文。
```

字段说明：

- `title`：文章标题，必填；
- `date`：发布日期，必须写成带引号的 `"YYYY-MM-DD"`；
- `slug`：网址最后一段，不能包含 `/`、`\`、`?` 或 `#`；
- `tags`：至少一个标签，一篇文章可以属于多个分类；
- `card`：主页动态卡片预设，可选；
- `excerpt`：主页和文章列表显示的摘要，必填；
- `draft`：为 `true` 时不发布，改为 `false` 或删除这一行后发布。

系统会根据日期和 slug 自动生成网址，例如 `/2026/07/23/my-ai-note/`。

### 7.2 选择动态卡片

| card 值 | 视觉与用途 | 推荐文章 |
|---|---|---|
| `code` | 动态终端、扫描线与光标 | C/C++、代码教程、编程笔记 |
| `cs` | 芯片、逻辑线路与信号流 | CS 基础、系统、数据结构 |
| `ai` | 神经网络节点与流动脉冲 | AI、机器学习、LLM |
| `vlog` | 取景框、风景层次与播放轴 | Vlog、摄影、旅行、生活 |
| `talk` | 对话气泡与输入动画 | 闲谈、随笔、日记、想法 |
| `web` | Client/Server 请求流 | Web、HTTP、前端 |
| `network` | 图节点与连线流动 | 图论、网络、算法 |
| `timeline` | 时间轴与阶段信号 | 经历总结、比赛记录、成长记录 |

在 Markdown 顶部写入例如 `card: vlog` 即可选择。如果不写 `card`，系统会根据 `tags` 自动匹配：`AI` 匹配 AI 卡片，`VLOG` 或 `生活` 匹配影像卡片，`闲谈` 或 `随笔` 匹配对话卡片，`WEB` 匹配请求流卡片。

### 7.3 原有 JSON 文章

原有正文继续保存在：

```text
content/articles.json
```

每篇文章结构如下：

```json
{
  "path": ["2026", "07", "21", "example"],
  "title": "示例文章",
  "date": "2026-07-21",
  "tags": ["WEB", "随笔"],
  "excerpt": "文章摘要",
  "content": "<h2 id=\"section\">正文标题</h2><p>正文内容</p>"
}
```

字段说明：

- `path`：完整年份、月份、日期和文章别名，同时决定文章网址；
- `title`：文章标题；
- `date`：发布日期，格式为 `YYYY-MM-DD`；
- `tags`：标签数组，一篇文章可以属于多个分类；
- `excerpt`：SEO 描述和列表摘要；
- `content`：HTML 格式正文。

### 7.4 修改现有文章

1. 新 Markdown 文章在 `content/posts/` 找到对应文件；原有文章仍在 `content/articles.json`；
2. 修改 `title`、`date`、`tags`、`excerpt` 或 `content`；
3. 运行 `npm.cmd run check`；
4. 运行 `npm.cmd run build`；
5. 打开文章页检查标题、目录、代码块和链接。

### 7.5 新增文章

新文章只需复制 Markdown 模板、填写顶部信息并撰写正文。系统会自动完成：

- 按发布日期排序；
- 生成文章编号；
- 生成文章网址；
- 更新主页前三篇；
- 更新“查看全部 N 篇”；
- 更新所有文章页的总数和大号数字；
- 根据 `tags` 创建分类；
- 更新主题总数；
- 更新每个分类的文章数量和文章列表；
- 为新年份生成文章页面。

新增后仍必须运行类型检查和生产构建。文章数据缺少路径、标题、日期、标签或正文时，构建会直接报错。

### 7.6 新年份文章

当前通用文章路由是：

```text
app/[year]/[...slug]/page.tsx
```

因此新增任意年份的文章都不需要再创建路由。例如：

```json
"path": ["2027", "01", "03", "new-post"]
```

会自动生成：

```text
/2027/01/03/new-post/
```

### 7.7 编辑原有 JSON 的注意事项

- 每个对象之间必须有英文逗号；
- JSON 字符串必须使用英文双引号；
- 正文中的双引号要写成 `\"`；
- 不要在最后一个对象后留下多余逗号；
- 中文文件必须保存为 UTF-8；
- 不要直接使用 Word 编辑 JSON。

修改后可运行以下命令检查 JSON 是否有效：

```powershell
node -e "JSON.parse(require('fs').readFileSync('content/articles.json','utf8')); console.log('文章 JSON 正常')"
```

### 7.8 代码块语言与复制

文章详情页会自动为 Markdown 围栏代码块和历史 HTML 代码块添加工具栏，工具栏显示语言名称和“复制”按钮。复制内容只包含代码正文，不包含工具栏或历史代码块的行号。

Markdown 文章应优先在三个反引号后明确填写语言，例如：

````markdown
```ts
const message: string = 'Hello, Youngkx';
console.log(message);
```
````

明确填写的语言标记优先级最高。未填写时，`ArticleView.tsx` 会根据类名和代码特征识别常见的 C、C++、C#、JavaScript、TypeScript、Python、Java、Rust、Go、HTML、CSS、JSON、Shell、PowerShell、SQL、YAML、Markdown 和 HTTP；无法可靠判断时显示 `Plain text`。

修改代码块增强功能时需要同时验证：

- Markdown 新文章的 `<pre><code class="language-*">` 结构；
- `content/articles.json` 中的历史 `<figure class="highlight ...">` 结构；
- 复制后的换行、缩进和特殊字符；
- 亮色、暗色和手机窄屏下的工具栏布局；
- 浏览器拒绝 Clipboard API 时的兼容复制路径。

## 8. 修改分类

分类由文章的 `tags` 自动生成。例如：

```json
"tags": ["OI", "算法"]
```

这篇文章会同时出现在 `OI` 和 `算法` 分类中。首次使用的新标签会自动出现在主页分类区和分类页，分类编号、锚点及文章数量也会自动生成。

`OI`、`C`、`WEB` 已在 `lib/articles.ts` 中配置了专用显示名称和说明。未知标签会直接使用标签文字作为标题，并显示“标签名 相关文章”。如果需要给新标签编写更正式的描述，可在 `lib/articles.ts` 的 `categoryDetails` 中增加可选配置；不配置也不会影响自动分类。

## 11. 修改动画、全站星空和样式

### 11.1 全站星空背景与主页星球

文件：

```text
components/Scene.tsx
```

这里控制全站 Three.js 背景粒子、指针光尘和点击反馈，以及仅在主页显示的星球、光线和随滚动变化的位置。

当前交互包括：

- 背景粒子自动漂浮、呼吸闪烁并随页面滚动改变位置；
- 粒子场由 `SiteExperience.tsx` 作为全站固定背景挂载；主页显示星球，文章、分类、正文和其他页面只显示粒子；
- 鼠标靠近时，附近粒子形成轻微涡流并增强亮度；
- 桌面端移动鼠标时产生短暂光尘轨迹；
- 桌面端点击时只产生少量光尘和轻微粒子扰动，不再显示大尺寸扩散环；
- 进入文章区域时自动降低粒子亮度，避免干扰阅读。

粒子色板应保持接近真实星空：以冷白、蓝白和钢蓝为主，只使用少量暖白表现恒星色温差异。不要加入高饱和紫色、绿色、粉色或彩虹色粒子。

桌面端和手机端使用不同的粒子数量、几何细分与设备像素比。手机端默认关闭触摸轨迹和点击反馈，但保留完整动画帧率。不要只在大屏设备上测试，也不要一次大幅提高粒子数量、几何细分或设备像素比，否则可能增加手机 GPU 负担。

修改后重点测试：

- 首页首屏；
- 鼠标移动、点击和触摸交互；
- 文章区域文字是否仍清晰；
- 分类区域；
- 手机端性能；
- 亮色和暗色主题；
- 系统开启“减少动态效果”后的静态表现。

### 11.2 页面动画

主页 Framer Motion 动画主要位于 `components/HomePage.tsx`。首屏使用一个高于视口的滚动舞台和固定在视口内的内容层：

- 页面初始状态只显示大号 `Youngkx Blog` 标题和背景星尘，不显示大星球；
- 用户向下滚动时，同一个标题根据首屏局部滚动进度保持清晰地缩小并迁移到第二阶段左侧，不对 `Youngkx Blog` 标题添加景深模糊；
- 第二阶段标题的横向位置根据标题实际宽度和简介内容栏左边缘动态计算，调整字号或缩放时仍需保持两者左对齐；
- 星球从屏幕边缘沿弧线旋转飞入，引句、说明和操作入口作为独立层从下方错峰上浮；
- 引句内部轮播使用从左向右的横向扫入和滑出，以区别于外层文字组的纵向景深动画；
- 引句、说明和操作入口全部进入后会完整同屏停留，继续滚动时再分层向上虚化退出；
- 文章、分类和内页的大标题及主要内容块同样采用由模糊到清晰的景深浮入，不增加文字外框或遮盖板；
- 滚动舞台结束后自然进入文章区域；
- 整个过程由用户滚动驱动，不使用定时器自动播放，也不使用全屏遮罩。

全站主题和页面过渡位于 `components/SiteExperience.tsx`：

- 主题切换时由独立全屏 Canvas 复制真实星空色板，高密度星尘沿随机非线性曲线高速乱飞并形成短暂遮盖，在遮盖最完整时更新主题色；
- 扰流遮盖使用比常驻背景更高的临时粒子密度、更大的曲线偏移和更长拖尾，但不得直接修改常驻 Three.js 粒子的数量与运动参数；
- 遮盖完成后，切换层星尘沿新的曲线继续逸散并淡出，显露已经更新的界面；
- 切换层与 `Scene.tsx` 中的 Three.js 背景粒子互相独立，不得为了切换动画修改、暂停或重建原有背景粒子场；
- Three.js 场景通过颜色插值跟随主题，不因切换主题重新初始化；
- 跨页面的站内链接复用同一套星尘扰流遮盖，同页锚点仍保持直接滚动；
- 开启“减少动态效果”时，主题和页面均立即切换，不播放粒子幕。

调整时需要同时检查：

- 页面刚打开时标题是否完整位于视口中心；
- 首次滚动时标题是否平滑退出，没有突然跳动；
- 简介与标题的显示区间是否自然衔接；
- 文章区域是否在滚动舞台结束后正常出现；
- 亮暗主题下标题描边和正文对比度是否足够；
- 手机竖屏下大标题是否被裁切；
- `prefers-reduced-motion` 开启时标题和简介是否直接按普通文档流显示。

常见参数：

- `duration`：动画持续时间；
- `delay`：延迟；
- `stiffness`：弹簧响应速度；
- `damping`：弹簧阻尼；
- `whileInView`：滚动到元素时触发。

不要一次把弹簧参数调得过大，否则页面会抖动或消耗更多性能。

### 11.3 颜色、字体和响应式布局

文件：

```text

```

修改后至少检查：

- 1440px 桌面宽度；
- 768px 平板宽度；
- 390px 手机宽度；
- 亮色主题；
- 暗色主题；

## 12. 域名与重定向

Cloudflare 配置位于：

```text
wrangler.jsonc
```

当前绑定：

```text
youngkx.cn
www.youngkx.cn
```

重定向代码位于：

```text
worker.mjs
```

当前逻辑会把：

```text
https://youngkx.cn/任意路径?任意参数
```

永久重定向到：

```text
https://www.youngkx.cn/相同路径?相同参数
```

不要把 `www.youngkx.cn` 再重定向回 `youngkx.cn`，否则会出现无限循环。

### 12.1 Cloudflare DNS

当前域名使用 Cloudflare DNS。除非迁移服务商，不要随意更换域名服务器。

当前 Cloudflare Nameserver：

```text
anahi.ns.cloudflare.com
zeus.ns.cloudflare.com
```

如果域名突然无法访问，依次检查：

1. 域名是否到期；
2. Nameserver 是否仍是上面两项；
3. Cloudflare 中域名状态是否为 Active；
4. Worker `youngkxblog` 是否仍绑定两个自定义域名；
5. 最近一次部署是否成功。

## 15. 回滚方法

### 15.1 代码尚未提交

先运行：

```powershell
git status
git diff
```

确认只是误改后，再使用编辑器撤销。不要在不清楚影响时执行 `git reset --hard`。

### 15.2 已提交但尚未部署

推荐创建一个反向提交：

```powershell
git log --oneline
git revert 提交编号
git push
```

`git revert` 会保留完整历史，比强制覆盖主分支安全。

### 15.3 已部署到 Cloudflare

有两种方式：

1. 在 Cloudflare 控制台的 Worker 部署记录中选择上一个正常版本并回滚；
2. 在 GitHub 对错误提交执行 `git revert`，重新运行构建和 `wrangler deploy`。

回滚后重新检查根域名、`www`、文章、分类和 404。

## 16. 常见问题

### 16.1 `npm.ps1 cannot be loaded`

原因：PowerShell 禁止运行脚本。

解决：

```powershell
npm.cmd run dev
npm.cmd run build
```

### 16.2 `wrangler` 未登录

```powershell
npx.cmd wrangler login
```

完成浏览器授权后重新部署。

### 16.3 构建成功但线上没有变化

常见原因：只执行了 `git push`，没有部署 Cloudflare。

```powershell
npm.cmd run build
npx.cmd wrangler deploy
```

### 16.4 页面显示旧内容

1. 强制刷新；
2. 使用无痕窗口；
3. 检查 Cloudflare 最新 Version ID；
4. 确认部署命令是在正确项目目录执行；
5. 确认 `out/` 是本次修改后生成的。

### 16.5 文章页面 404

检查：

- `content/articles.json` 的 `key`；
- `HomePage.tsx` 中的 `href`；
- 中文标点和大小写；
- `path` 是否包含年份、月份、日期和文章别名；
- `npm.cmd run build` 输出中是否出现该文章。

### 16.6 根域名不跳转

检查：

- `worker.mjs` 是否仍包含 `youngkx.cn` 判断；
- `wrangler.jsonc` 是否包含两个自定义域名；
- 是否执行了最新的 `wrangler deploy`；
- DNS 是否仍由 Cloudflare 托管。

### 16.7 网站图标没有更新

检查：

- `app/icon.png` 是否存在；
- `app/apple-icon.png` 是否存在；
- `npm run build` 输出中是否包含图标路由；
- 是否重新部署；
- 浏览器是否仍在使用旧缓存。

### 16.8 中文乱码

所有源码、JSON 和 Markdown 文件都必须保存为 UTF-8。不要使用 ANSI 或 GBK 保存。

## 18. 最重要的维护原则

1. 修改前先拉取最新代码；
2. 一次只处理一类修改；
3. 不要直接编辑 `out/`；
4. 不要提交账号密码或 Token；

8. 删除文件前先确认它不是当前 Next.js 或 Cloudflare 配置的一部分；
9. 重要改动使用 Git 分支和 Pull Request；
10. 出现问题优先回滚，不要在正式站点上连续试错。
